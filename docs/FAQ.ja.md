# ClawClip FAQ / よくある質問

> `v1.1.x`（`v1.1.0` を含む）向け。よく聞かれる質問にだけ短く答える FAQ です。

- 正確な対応範囲を確認したい場合は [COMPATIBILITY.md](./COMPATIBILITY.md)
- セットアップやセルフホスティングの詳細は [DEPLOYMENT.md](./DEPLOYMENT.md)

## なぜ Demo データしか表示されないのですか？

ClawClip は、互換性のある実際のローカルセッションをまだ見つけていないとき、内蔵の Demo データを表示します。公開ライブデモも意図的にサンプルデータだけです。

- 自分の実行結果を見たいなら、ローカル環境かセルフホスト環境を使ってください。
- まずいくつかタスクを実行し、そのあと `agents/<agent>/sessions/*.jsonl` または `<root>/sessions/*.jsonl` を含むフォルダを ClawClip に指定するのがいちばん確実です。
- それでも Demo のままなら、下の **「ページは開くのにセッションが表示されないのはなぜですか？」** を見てください。

## ClawClip にカスタムのログディレクトリをスキャンさせるには？

単一のトランスクリプトファイルではなく、**データルート** を ClawClip に指定してください。

```bash
OPENCLAW_STATE_DIR=/path/to/.openclaw
CLAWCLIP_LOBSTER_DIRS=/data/runs;/data/export
CLAWCLIP_SESSION_EXTENSIONS=.jsonl,.ndjson
```

- `OPENCLAW_STATE_DIR`: 既定の OpenClaw state ルートを置き換えます。
- `CLAWCLIP_LOBSTER_DIRS`: 追加のルートを 1 つ以上指定します。区切りはカンマまたはセミコロンです。
- `CLAWCLIP_SESSION_EXTENSIONS`: トランスクリプトの拡張子が `.jsonl` ではない場合にだけ必要です。
- 安全で分かりやすいレイアウトは `agents/<agent>/sessions/*.jsonl` または `<root>/sessions/*.jsonl` です。
- Docker では、これらのパスは **ホスト側ではなくコンテナ内のパス** である必要があります。

## OpenClaw / ZeroClaw / Claw / custom JSONL は実際どこまで対応していますか？

短く言うと、**OpenClaw と ZeroClaw が主対象です。Claw と custom JSONL は best-effort であり、「何でも動く」全面対応ではありません。**

- もっとも手厚く対応しているのは、公式 OpenClaw / ZeroClaw 形式のローカル JSONL セッションレイアウトです。
- 互換性のあるトランスクリプトがあれば、ClawClip は `~/.claw` もスキャンします。
- パーサーはすでに、一般的な OpenClaw 系イベント、複数 `tool_calls`、`tool_result` / `function_call_output`、reasoning / thinking ブロック、そして一部の旧 chat-completions 風の行も扱えます。
- SQLite / `.db` / `.sqlite` を直接読む機能には **まだ** 対応していません。
- `sessions.json` はメタデータ補助には使えますが、**トランスクリプト本体ではありません**。
- 正確な境界を確認したい場合は [COMPATIBILITY.md](./COMPATIBILITY.md) を見てください。

## データは外部にアップロードされますか？

既定ではありません。セッションの検出、解析、リプレイ、スコアカード分析は、あなたのマシン上か、あなた自身のデプロイ環境内で動きます。

- ClawClip は、既定では Agent の実行データをアップロードしません。
- ネットワークを使う可能性があるのは公開価格の更新だけで、更新されるのは価格参照情報のみです。セッション内容は送信されません。
- ClawClip をセルフホストしている場合、データは **あなたが配置した場所** にとどまります。

## Docker / `npm start` / dev モードはそれぞれいつ使うべきですか？

やっていることに合う**最小構成**を選んでください。

- `npm start`: 自分のマシンやシンプルなサーバーで ClawClip を使う最速の方法です。
- Docker / `docker compose up --build`: 分離したデプロイ、分かりやすいボリュームマウント、運用しやすいサーバー構成が欲しいときに向いています。
- `npm run dev:server` + `npm run dev:web`: ClawClip 自体を開発する人向けです。
- `dev:server` だけを起動して `/` を開くと、ビルド済みフロントエンドがないため完全なアプリではなくバックエンドメッセージが出ることがあります。これは正常です。

より詳しいセットアップ手順は [DEPLOYMENT.md](./DEPLOYMENT.md) を見てください。

## ページは開くのに、なぜセッションが表示されないのですか？

ほとんどの場合、ClawClip 自体は正常で、互換性のあるトランスクリプトをまだ見つけていないだけです。

- 実際にタスクを実行し、トランスクリプトが生成されていることを確認してください。
- スキャンパスが 1 つの JSONL ファイルではなく、`agents/<agent>/sessions` の **ひとつ上のフォルダ** を指しているか確認してください。
- Docker では、ホスト側フォルダが正しくマウントされていて、環境変数が **コンテナ内パス** を指しているか確認してください。
- ランタイムが主に SQLite / DB に保存している場合は、まず JSONL をエクスポートまたは同期してください。
- 別の拡張子を使っているなら、`CLAWCLIP_SESSION_EXTENSIONS` を設定してください。
- 設定ファイルや `sessions.json` しかない場合、それだけではリプレイ可能なトランスクリプトにはなりません。

## スコアはなぜ「標準的な benchmark スコア」ではないのですか？

ClawClip は、1 つの共通テストセットで Agent を採点しているわけではないからです。Agent Scorecard は、あなた自身のローカル実行の振る舞いから作る **ヒューリスティックな診断** です。

- 使いどころは、変更前後の比較、改善の方向づけ、レビューの高速化です。
- ベンダーの benchmark 証明や、チーム横断の普遍的な順位表として扱わないでください。
- Demo だけのスコアやカーブはあくまで参考で、実セッションのほうがはるかに重要です。

## 価格データはリアルタイムですか？

厳密な意味での「ライブ課金ダッシュボード」ではありません。ClawClip には検証済みの静的フォールバック価格表が同梱されており、ネットワークが有効なら新しい公開価格の参照情報を更新できます。

- 使いどころは、コストの方向感や、最適化に見合う価値があったかの判断です。
- プロバイダーの請求書を 1 行ずつ争うための最終根拠にはしないでください。
- ごく新しいモデルや名前が特殊なモデルは、価格マッピングが追いつくまで一時的に推定値へフォールバックすることがあります。
