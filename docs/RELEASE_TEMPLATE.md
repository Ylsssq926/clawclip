# 发布文案模板

## 标准 Release 文案（中文）

```
v{VERSION} — {一句话说这版解决了什么}

{一个具体的场景，让人停一下}

这版：
- {功能1}：{一句话说用户感受}
- {功能2}：{一句话说用户感受}
- {功能3}：{一句话说用户感受}

装上，指向你的日志目录，15 秒看到第一张诊断报告。

→ {GitHub Release 链接}
```

## 标准 Release 文案（英文）

```

{one concrete scene that makes people stop}

- {feature1}: {one line on what users feel}
- {feature2}: {one line on what users feel}
- {feature3}: {one line on what users feel}

```

## 文案原则

1. 先给画面，再给解决方案
2. 用人会说的话，不要说明书腔
3. 宁可少一点，也别一上来写满
4. 不要"优化"、"分析"、"提供"、"实现"这类词
5. 一句话要能让人停一下，想接一句
